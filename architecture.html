Prompt Caching Strategy

System Block Caching

Each agent's ~800-token system prompt is marked cache_control: ephemeral. Cached for 5 minutes — identical across all user sessions for that agent.

Cost Reduction

System prompt costs drop ~90% after first cache hit. At scale (1000 users × 10 messages/day), saves ~$50/day in API costs vs uncached.

Context Injection

User's financial profile (income, accounts, goals) is appended after the cached block as dynamic context, personalizing each response.

Document Upload Flow
1

File Validation

MIME type check (PDF/JPG/PNG/CSV), 20MB size limit, malformed file rejection

2

Secure Storage

Stored in Supabase Storage bucket financial-docs under user's UUID prefix. Private RLS policy.

3

Claude Vision Extraction

PDF/images sent as base64 to Claude claude-sonnet-4-6 with structured extraction prompt. Returns typed JSON.

4

Auto-Import

Bank statement transactions auto-imported into Bookkeeper. Salary slip income added to financial profile.

5

AI Analysis Summary

Plain-language summary stored alongside structured JSON data for display in Document Library.

Security Architecture
Defense in Depth

Financial data requires the highest security standards. Every layer is hardened.

Authentication & Authorization
🔐

Supabase Auth (JWT)

Email/password + OAuth providers. JWT tokens with 1-hour expiry, auto-refresh via httpOnly cookies. No custom auth code.

🛡️

Row Level Security (RLS)

PostgreSQL RLS on all 12 tables. Data isolation enforced at DB level — bypass impossible even with compromised app code.

🔑

API Route Guards

Every API route validates supabase.auth.getUser() before processing. Server-side only — no client-side auth checks trusted.

🗝️

Secret Management

All keys in env vars only. ANTHROPIC_API_KEY, SUPABASE_SERVICE_ROLE_KEY, PLAID_SECRET never exposed to client. Vercel encrypted env vars in production.

Data & Document Security
📁

Private Storage Bucket

Supabase Storage bucket financial-docs is private. Files stored under {user_id}/{timestamp}_{filename}. RLS policy prevents cross-user access.

🔒

File Validation

MIME type whitelist (PDF/JPG/PNG/CSV only). 20MB size cap. Filename sanitization. Buffer read — no shell execution.

🏦

Plaid Token Security

Plaid access tokens stored encrypted in DB. Read-only Plaid scope requested. No payment/transfer permissions. Users can disconnect anytime.

🌐

Transport Security

TLS 1.3 enforced. HSTS headers. No HTTP fallback. CORS restricted to app domain. CSP headers prevent XSS.

AI Safety Guardrails

Scope Limitation

  • Agents stay in their domain (tax agent won't give mortgage advice)
  • No execution of financial transactions
  • Always recommend licensed advisor for complex cases

Disclaimer Enforcement

  • "Not a substitute for professional advice" in every chat
  • AI output clearly labelled as AI-generated
  • Canadian jurisdiction default with explicit US flag

Prompt Injection Defense

  • System prompts cached server-side — not user-modifiable
  • User context injected after cache boundary
  • Agent type validated on server — client can't switch agents

Deployment & Setup
From zero to production
Setup Checklist
  • Create Supabase project → run schema.sql + schema_documents.sql
  • Create Storage bucket financial-docs (private)
  • Add RLS storage policy for user-scoped access
  • Get Anthropic API key from console.anthropic.com
  • Create Plaid developer account → get sandbox keys
  • Copy .env.local.example → .env.local, fill all keys
  • npm install → npm run dev
  • Push to GitHub → connect to Vercel
  • Add all env vars to Vercel project settings
  • Enable Vercel Analytics in project settings
Environment Variables
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://xxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...
SUPABASE_SERVICE_ROLE_KEY=eyJ...
# Anthropic
ANTHROPIC_API_KEY=sk-ant-...
# Plaid (optional)
PLAID_CLIENT_ID=your_client_id
PLAID_SECRET=your_secret
PLAID_ENV=sandbox|development|production
# App
NEXT_PUBLIC_APP_URL=https://yourapp.vercel.app
Note: Plaid integration requires registering your app at plaid.com/docs. Sandbox mode works immediately. Production requires Plaid approval (~1-2 weeks). You can ship the full app without Plaid — the CSV upload + manual document import path works for all users.
